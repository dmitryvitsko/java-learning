# 🔷 Хранилище фигур для уроков рукоделия

## 📋 Задача
Создать хранилище геометрических фигур для учителя рукоделия.
Каждая фигура хранится в формате [N, a, b, c...] где N - количество углов.

---

## 🧠 Теория

### ArrayList<int[]> как хранилище фигур
```java
public static ArrayList<int[]> figures = new ArrayList<>();
```
Каждый элемент списка - массив int[].
Первый элемент массива - количество углов фигуры.
Остальные элементы - размеры сторон.

### Формат хранения фигур
```java
// Треугольник: 3 угла, стороны a, b, c
int[] tri = {3, a, b, c};

// Прямоугольник: 4 угла, ширина и высота
int[] rect = {4, width, height};

// Пятиконечная звезда: 5 углов, 5 сторон
int[] star = {5, l0, l1, l2, l3, l4};
```
Первый элемент всегда количество углов - универсальный формат
для любой фигуры с любым количеством сторон.

### Статическое поле класса
```java
public static ArrayList<int[]> figures = new ArrayList<>();
```
Поле объявлено как static - доступно из статических методов
без создания объекта класса Main.

### Вывод через запятую без лишней запятой
```java
System.out.print(figure[0]);
for (int i = 1; i < figure.length; i++) {
    System.out.print(",");
    System.out.print(figure[i]);
}
System.out.println();
```
Первый элемент выводим отдельно, остальные с запятой перед ними.

---

## 🛠 Что использовал
- `ArrayList<int[]>` - список массивов как гибкое хранилище
- Статические методы для добавления каждого типа фигуры
- Единый формат [N, стороны...] для любой фигуры
- for-each + for для вывода

---

## 💡 Вывод
Один универсальный формат int[] позволяет хранить фигуры
с разным количеством сторон в одном списке.
Это гибче чем создавать отдельный класс для каждой фигуры -
хотя в реальном проекте классы были бы правильным решением.
