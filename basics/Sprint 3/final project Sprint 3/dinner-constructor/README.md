# 🍽 Конструктор обедов — Финальная задача Спринт 3

## 📋 Задача
Консольное приложение для генерации случайных комбинаций блюд.
Пользователь добавляет блюда по типам, затем генерирует случайные обеды.

---

## ⚙️ Функциональность
- Добавить новое блюдо с указанием типа (Суп, Салат, Горячее...)
- Сгенерировать N случайных комбинаций из выбранных типов блюд
- Проверка что тип блюда существует в хранилище

---

## 🏗 Архитектура — 2 класса

### `Main.java`
Точка входа. Консольное меню, считывание ввода пользователя,
вызов методов DinnerConstructor.

### `DinnerConstructor.java`
Вся логика приложения:
- Хранилище блюд по типам через HashMap
- Генерация случайных комбинаций через Random
- Проверка существования типа блюда

---

## 🧠 Теория

### HashMap как хранилище с группировкой
```java
HashMap<String, ArrayList<String>> dinnersByType = new HashMap<>();
```
Ключ - тип блюда ("Суп", "Салат").
Значение - список блюд этого типа.
Тот же паттерн что и в финансовом приложении с категориями трат.

### Random для случайного выбора
```java
Random random = new Random();
int dishIndex = random.nextInt(numberOfDishesForType);
String selectedDish = availableDishes.get(dishIndex);
```
nextInt(n) возвращает случайное число от 0 до n-1.
Используем его как индекс для случайного выбора из списка.

### ArrayList<ArrayList<String>> - список списков
```java
ArrayList<ArrayList<String>> combos = new ArrayList<>();
```
Каждая комбинация это список блюд (ArrayList<String>).
Все комбинации хранятся в списке этих списков.
Вложенная структура данных - следующий уровень после простых списков.

### scanner.nextLine() везде
```java
String command = scanner.nextLine();
```
В этом проекте везде nextLine() - нет проблем с буфером
которые возникают при смешивании nextInt() и nextLine().

## 🛠 Что использовал
- `HashMap<String, ArrayList<String>>` - хранилище блюд по типам
- `ArrayList<ArrayList<String>>` - список комбинаций
- `Random.nextInt()` - случайный выбор блюда
- `scanner.nextLine()` - единый способ считывания ввода
- `checkType()` - валидация типа перед добавлением
- `package ru.practicum.dinner` - первое знакомство с пакетами
