# 💰 Финансовое приложение v9

## 📋 Задача
Переход от массива трат к ArrayList. Добавлены команды очистки
списка и поиска/удаления траты по уникальному номеру транзакции.
Введён класс-контейнер Expense.

---

## 🧠 Теория

### Зачем класс-контейнер вместо простого double
```java
public class Expense {
    private double value;
    private int transaction; // уникальный номер
}
```
Сравнивать траты просто по сумме (double) ненадёжно -
могут быть две одинаковые траты на одну сумму в разные дни.
Поэтому каждая трата получает уникальный номер транзакции
при создании - это позволяет точно находить нужный элемент.

### Инкапсуляция через геттеры и сеттеры
```java
private double value; // приватное поле

public double getValue() {
    return value;
}
public void setValue(double val) {
    value = val;
}
```
Поля закрыты модификатором private - доступ к ним только
через публичные методы. Это защищает данные от случайного
изменения извне.

### ArrayList<Expense> вместо ArrayList<double>
```java
ArrayList<Expense> expenses = new ArrayList<>();
expenses.add(new Expense(expense));
```
Список теперь хранит не просто числа, а объекты с
дополнительной информацией - суммой и номером транзакции.

### Поиск элемента по условию с прерыванием цикла
```java
void removeExpense(int transaction) {
    Expense currentExp = null;
    for (Expense exp : expenses) {
        if (exp.getTransaction() == transaction) {
            currentExp = exp;
        }
    }
    if (currentExp == null) {
        System.out.println("Такой траты нет.");
    } else {
        expenses.remove(currentExp);
    }
}
```
Перебираем список, сохраняем найденный объект в переменную.
Если после цикла переменная всё ещё null - элемент не найден.

---

## 🛠 Что использовал
- Класс-контейнер `Expense` с приватными полями
- Геттеры и сеттеры для инкапсуляции
- `ArrayList<Expense>` вместо массива примитивов
- Поиск объекта по уникальному полю transaction
- `expenses.clear()` для полной очистки списка
- `expenses.remove(object)` для удаления найденного объекта

---

## 💡 Вывод
Когда объекты нужно различать однозначно - сумма недостаточна,
нужен уникальный идентификатор. Класс-контейнер с собственным
ID - стандартный паттерн для таких задач, используется
повсеместно в реальных приложениях (ID пользователя, ID заказа и т.д.)
