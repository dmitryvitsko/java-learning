# 🍲 SOUPER-3000 — Конструкторы и наследование в Java

## 📖 Описание проекта

В этом задании реализована модель робота-повара **SOUPER-3000**, который умеет готовить различные виды супов.

Основная задача — научиться использовать **наследование** и **конструкторы** для создания объектов разными способами.

В проекте реализована иерархия классов, где каждый следующий суп наследует свойства предыдущего и может создаваться:
- с нуля;
- на основе уже готовой суповой основы;
- на основе готового бульона.

---

## 🏗 Структура проекта

```
SoupBase
├── Bouillon
│   └── PotatoSoup
└── VegetarianSoup
```

### Классы

- **SoupBase** — базовая основа любого супа (вода + соль).
- **Bouillon** — бульон, который наследует суповую основу и добавляет мясо.
- **VegetarianSoup** — вегетарианский суп на основе суповой основы.
- **PotatoSoup** — картофельный суп, наследуется от бульона и добавляет картофель и морковь.

---

## ⚙️ Что реализовано

- использование наследования (`extends`);
- вызов конструкторов родительских классов через `super()`;
- перегрузка конструкторов;
- создание объектов несколькими способами;
- переопределение метода `printIngredients()`.

---

## 📚 Что было изучено

Во время выполнения задания удалось разобраться с несколькими важными принципами ООП:

- наследование классов;
- работа конструктора родительского класса через `super()`;
- почему при наследовании нельзя обращаться к полям родителя до вызова его конструктора;
- перегрузка конструкторов;
- повторное использование уже созданных объектов для построения новых.
