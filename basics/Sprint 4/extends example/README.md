# 🔷 Фигуры — первое знакомство с наследованием

## 📋 Задача
Разобраться в иерархии классов и понять как работает наследование.
Загадка: у трёх сыновей один родитель, у одного родителя один сын,
а один сын сам родитель.

---

## 🧩 Разгадка загадки
- "У трёх сыновей один родитель" → Circle, Square, Rectangle наследуют Figure
- "У одного родителя один сын" → Circle является родителем для Ellipse
- "Один сын сам родитель" → Circle одновременно сын Figure и родитель Ellipse

---

## 🧠 Теория

### Наследование через extends

```java
class Circle extends Figure { }
```

Circle получает все методы Figure автоматически - drawCircle(), drawSquare(), drawRectangle(). Не нужно их переписывать.

### Цепочка наследования

```java
class Ellipse extends Circle { }
```

Ellipse наследует Circle, который наследует Figure. Значит Ellipse получает методы и от Circle и от Figure. Поэтому ellipse.drawCircle() работает - метод пришёл через цепочку Figure → Circle → Ellipse.

### Свой метод у наследника

```java
class Ellipse extends Circle {
    public void circleToEllipse() {
        System.out.println("Превращаем круг в овал");
    }
}
```

Ellipse добавляет свой уникальный метод которого нет у родителей. У Figure и Circle нет circleToEllipse() - только у Ellipse.

---

## 🛠 Что использовал

- `extends` - ключевое слово для наследования
- Цепочка наследования Figure → Circle → Ellipse
- Наследование методов родительского класса
- Добавление собственного метода в класс-наследник

---

## 💡 Вывод

Наследование позволяет не дублировать код. Общие методы пишутся один раз в родителе - все наследники получают их автоматически. Это один из главных принципов ООП.
