# ♻️ Recycling Chat Bot

Консольный чат-бот для компании по переработке вторсырья.

Проект выполнен в рамках обучения на курсе **«Java-разработчик»** от **Яндекс Практикума** и посвящён изучению **enum**, **HashMap**, **ArrayList**, **switch-case** и организации логики через отдельные классы.

---

## 📌 Описание проекта

Компания «Собираем и перерабатываем» проводит экологическую акцию и начисляет бонусные баллы за сдачу вторсырья.

Чат-бот позволяет:

- посмотреть карту пунктов приёма вторсырья;
- проверить по коду на упаковке, подходит ли материал для переработки;
- рассчитать количество бонусных баллов за сданное вторсырьё.

---

## ⚙️ Функциональность

### 🗺 Карта пунктов приёма

Команда:

```text
MAP
```

Выводит ссылку на карту пунктов приёма вторсырья.

---

### 🔍 Проверка перерабатываемости материала

Команда:

```text
RECYCLABILITY
```

Позволяет ввести код переработки и получить информацию:

- описание материала;
- подходит ли он для переработки;
- в какой контейнер его нужно поместить.

Пример:

```text
Введите код переработки:
1
Это полиэтилентерфталат. Положите его в контейнер с типом «Пластик».
```

---

### 🎁 Расчёт бонусных баллов

Команда:

```text
BONUS
```

Программа рассчитывает бонусы по правилам акции:

- меньше 10 кг → коэффициент 10;
- 10 кг и больше → коэффициент 15.

Пример:

```text
Введите количество вторсырья, кг:
12
Количество бонусных баллов: 180.
```

## 🎯 Чему научился

В этом проекте особенно понравилось, как удобно `enum` и `HashMap` помогают моделировать реальные бизнес-сценарии.

- `ChatCommand` делает команды бота типобезопасными.
- `MaterialType` позволяет избежать ошибок со строками.
- `HashMap<MaterialType, String>` удобно связывает тип материала с названием контейнера.
- `ArrayList<RecyclableMaterial>` хранит базу материалов для поиска по коду.

Постепенно начинаю понимать, почему в Java так много внимания уделяется структуре данных и типобезопасности — код становится гораздо понятнее и надёжнее.
