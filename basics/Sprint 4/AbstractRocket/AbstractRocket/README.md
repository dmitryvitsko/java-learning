# 🚀 AbstractRocket

Учебный проект по Java, посвящённый изучению **абстрактных классов**, **наследования** и **шаблонного метода (Template Method)**.

## 📌 Описание

В проекте реализована система запуска космических ракет.

Существует единая последовательность запуска, которая не может изменяться:

1. Проверка систем.
2. Продувка магистралей.
3. Отсоединение наземных коммуникаций.
4. Зажигание двигателей.
5. Старт ракеты.

Эта последовательность описана в абстрактном классе `AbstractRocketLaunch` и защищена от изменений.

При этом каждая модель ракеты имеет свои особенности:

- **Союз-2Б** — выполняет проверку сразу двух бортовых компьютеров.
- **Протон-М** — состоит из нескольких модулей, поэтому проверка, продувка и зажигание выполняются для каждого блока.
- **Falcon 9** — после взлёта автоматически выполняет ориентацию ракеты вдоль вращения Земли.
- **Starship** — имеет 33 двигателя и выполняет предварительную проверку двигателей ещё до запуска, а после старта также проходит процедуру ориентации.

Все особенности реализованы без изменения общего алгоритма запуска, исключительно за счёт наследования и переопределения методов.

## ⚙️ Что реализовано

- создание абстрактного класса;
- использование абстрактных методов;
- наследование;
- переопределение поведения в дочерних классах;
- реализация различных сценариев запуска для разных моделей ракет;
- сохранение общего алгоритма запуска без его изменения (Template Method).

## 🛠️ Используемые технологии

- Java
- ООП
- Абстрактные классы
- Наследование
- Полиморфизм
- Template Method Pattern

## 🎯 Чему я научился

Во время выполнения задания я понял, зачем вообще нужны абстрактные классы.

Вместо того чтобы копировать один и тот же алгоритм в разные классы, можно один раз описать общий процесс в базовом классе, а различия вынести в наследников.

Такой подход позволяет расширять программу, не меняя уже работающий код, что делает её более безопасной, понятной и удобной для сопровождения.
