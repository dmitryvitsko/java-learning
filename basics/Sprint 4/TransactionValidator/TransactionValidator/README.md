# 💳 TransactionValidator

Консольное приложение для проверки суммы банковского перевода.

Проект выполнен в рамках обучения на курсе **«Java-разработчик»** от **Яндекс Практикума** и посвящён изучению **статических методов**, **констант** и базовой валидации данных.

---

## 📌 Описание проекта

Программа имитирует часть логики банковского приложения.

Пользователь вводит сумму перевода, после чего система проверяет:

- не меньше ли сумма минимально допустимой;
- не превышает ли сумма установленный лимит.

Если сумма корректна — перевод считается успешно выполненным.

---

## ⚙️ Функциональность

- ввод суммы перевода через консоль;
- проверка минимальной суммы перевода;
- проверка максимальной суммы перевода;
- вывод сообщений об ошибке;
- подтверждение успешного перевода.

---

## 🛠 Используемые технологии

- Java
- Scanner
- static методы
- final константы
- условные конструкции (`if / else`)

## 💻 Пример работы программы

### Успешный перевод

```text
Пожалуйста, введите сумму перевода в рублях.
1500
Спасибо! Ваш перевод на сумму 1500.0 р. успешно выполнен.
```

### Сумма меньше минимальной

```text
Пожалуйста, введите сумму перевода в рублях.
0
Минимальная сумма перевода: 1 р. Попробуйте ещё раз!
```

### Сумма превышает лимит

```text
Пожалуйста, введите сумму перевода в рублях.
10000
Максимальная сумма перевода: 5000 р. Попробуйте ещё раз!
```

## 🎯 Чему научился

В этом проекте впервые использовал **константы** для хранения бизнес-ограничений (`MIN_AMOUNT` и `MAX_AMOUNT`) и вынес проверку суммы в отдельный класс `TransactionValidator`.

Такой подход делает код более читаемым, удобным для поддержки и позволяет изменять правила проверки в одном месте, не затрагивая остальную часть программы.
