# 👷 Калькулятор рабочих часов

## 📋 Задача
Доработать калькулятор рабочих часов и расставить
модификаторы доступа в соответствии с логикой приложения.

---

## 🧠 Теория

### Модификаторы доступа
```java
private int workingHours = 8;    // только внутри класса
private String workerName;       // только внутри класса
public double coefficient;       // доступен отовсюду
```

Три уровня которые использовали в этой задаче:

- `private` - поле доступно только внутри своего класса.
  Никто снаружи не может прочитать или изменить напрямую.
- `public` - доступно отовсюду. Любой класс может обратиться.
- (без модификатора) - package-private, доступно внутри пакета.

### Зачем закрывать поля?
```java
private int workingHours = 8;
private String workerName;
```
Рабочие часы и имя работника - это данные которые
не должны меняться извне. Закрываем через private.
Если нужно получить значение - используем геттер:
```java
public String getWorkerName() {
    return workerName;
}
```
Геттер публичный - читать можно, но изменить нельзя.
Это и есть инкапсуляция.

### Публичный конструктор
```java
public WorkCalculator(String newName) {
    workerName = newName;
}
```
Конструктор public - иначе из другого пакета
не получится создать объект класса.

### Пакеты и модификаторы
```java
package ru.yandex.practicum;        // Main.java
package ru.yandex.practicum.work;   // WorkCalculator.java
```
Классы в разных пакетах - поэтому private поля
WorkCalculator недоступны из Main даже при желании.

---

## 🛠 Что использовал
- `private` - для workingHours и workerName
- `public` - для coefficient, конструктора и всех методов
- Геттер `getWorkerName()` для чтения приватного поля
- `System.out.printf()` для форматированного вывода с %.2f
- Два пакета: ru.yandex.practicum и ru.yandex.practicum.work

---

## 💡 Вывод
Инкапсуляция - это не просто слово из теории ООП.
Это конкретный инструмент: закрываешь данные через private,
открываешь доступ только через методы которые ты контролируешь.
Так данные защищены от случайного изменения извне.
