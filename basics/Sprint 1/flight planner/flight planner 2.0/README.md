# ✈️ Планировщик авиамаршрута v2

## 📋 Задача
Улучшенная версия планировщика. Добавлены зимние месяцы в список
"остаться в Москве", булевы переменные для условий, упрощено ветвление.

---

## 🧠 Теория

### Булевы переменные для условий
Вместо длинных условий прямо в if - сохраняем в переменные:
```java
// Было:
if (ticketMoscowParis < ticketMoscowLondonParis && britainVisa == 1)

// Стало - читается как обычный текст:
boolean directTicketsCheaper = ticketMoscowParis < ticketMoscowLondonParis;
boolean hasBritainVisa = britainVisa == 1;

if (!directTicketsCheaper && hasBritainVisa)
```
Код становится самодокументируемым - не нужно объяснять что проверяется.

### Оператор отрицания !
```java
!directTicketsCheaper // НЕ дешевле = дороже или равно
!hasBritainVisa       // НЕТ визы
```

### Сезон по умолчанию
```java
String season = "Зима"; // задаем сразу
if (...) season = "Весна";
else if (...) season = "Лето";
else if (...) season = "Осень";
// Зима остается если ни одно условие не сработало
```
Не нужно отдельно проверять зиму - она уже задана по умолчанию.

---

## 🛠 Что использовал
- Булевы переменные `boolean` для хранения условий
- Оператор отрицания `!`
- Сезон по умолчанию без лишнего ветвления

---

## 💡 Вывод
Булевы переменные с говорящими именами делают код читаемым.
`!directTicketsCheaper && hasBritainVisa` читается почти как обычная фраза.
