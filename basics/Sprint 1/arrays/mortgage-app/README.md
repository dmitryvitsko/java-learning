# 🏦 Ипотечное приложение

## 📋 Задача
Программа проверяет соответствие количества документов и участников сделки,
и выводит список документов для каждого участника.

---

## 🧠 Теория

### Длина массива
```java
String[] participants = {"Пётр", "Надежда", "Валентин"};
participants.length // вернет 3
```
`.length` - свойство массива, не метод. Без скобок!

### Перебор массива по индексу
```java
for (int i = 0; i < participants.length; i++) {
    System.out.println(participants[i]);
}
```
Условие `i < participants.length` - не `<=`, иначе выйдем за границу массива
и получим `ArrayIndexOutOfBoundsException`.

### Сравнение длин двух массивов
```java
if (participants.length == documents.length) {
    // всё ок
} else {
    // ошибка
}
```
Классический паттерн валидации данных перед обработкой.

---

## 🛠 Что использовал
- Массив `String[]`
- Свойство `.length`
- Цикл `for` с индексом для перебора двух массивов одновременно

---

## 💡 Вывод
Проверка длин массивов перед обработкой - базовая защита от ошибок.
В реальных проектах такая валидация обязательна.
