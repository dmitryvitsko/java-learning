# 📊 Проверка расходов по индексу

## 📋 Задача
Программа заполняет массив расходов за неделю и позволяет
пользователю проверить расходы за любой день по индексу.
Защита от выхода за границы массива.

---

## 🧠 Теория

### ArrayIndexOutOfBoundsException
Самая частая ошибка при работе с массивами - обращение по несуществующему индексу:
```java
int[] arr = new int[7]; // индексы от 0 до 6
arr[7]  // ошибка - такого индекса нет
arr[-1] // ошибка - отрицательный индекс
```

### Защита от выхода за границы
```java
if (day < 0) {
    // слишком маленький индекс
} else if (day >= expenses.length) {
    // слишком большой индекс
} else {
    // всё ок - работаем с массивом
    break;
}
```
Проверяем ОБЕ границы - снизу и сверху.

### Паттерн: цикл с валидацией
```java
while (true) {
    // просим ввод
    // если ввод неверный - продолжаем цикл
    // если верный - делаем работу и break
}
```
Стандартный паттерн когда нужно повторять запрос до корректного ввода.

---

## 🛠 Что использовал
- Бесконечный цикл `while (true)`
- `break` при корректном вводе
- Проверка границ массива перед обращением
- `expenses.length - 1` для вывода максимального индекса

---

## 💡 Вывод
Всегда валидируй индекс перед обращением к массиву.
`while (true) + break` - удобный паттерн для повторного запроса ввода.
