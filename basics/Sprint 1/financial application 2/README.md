# 🍕 Советник по ужину v2

## 📋 Задача
Программа советует где поужинать с учетом двух факторов:
- остаток денег до зарплаты
- количество дней до зарплаты

---

## 🧠 Теория

### Вложенные условия
Условия можно вкладывать друг в друга - проверять второе условие только если первое выполнилось.

```java
if (внешнее условие) {
    if (внутреннее условие) {
        // выполнится только если оба условия true
    }
}
```

### Упрощение else if
В цепочке `else if` не нужно повторять уже проверенное условие:

```java
// Лишнее - так не надо:
} else if (money >= 3000 && money < 10000) {

// Правильно - так лаконичнее:
} else if (money < 10000) {
```
Если дошли до `else if` - предыдущее условие уже не выполнилось, значит нижняя граница гарантирована.

---

## 🛠 Что использовал
- `Scanner` - считывание нескольких значений от пользователя
- `scanner.nextDouble()` - для дробного числа
- `scanner.nextInt()` - для целого числа
- Вложенные `if/else` - проверка нескольких условий одновременно

---

## 💡 Вывод
Чем больше вложенность условий - тем важнее следить за читаемостью.
Если вложенность больше 2-3 уровней - это сигнал что код стоит переписать.
