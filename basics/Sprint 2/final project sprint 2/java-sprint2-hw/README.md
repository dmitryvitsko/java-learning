# 🏃 Счётчик калорий — Финальный проект Спринт 2

## 📋 О проекте
Консольное приложение для отслеживания шагов, калорий и статистики активности.
Финальный проект второго спринта курса Яндекс Практикум Java-разработчик.

---

## 🎯 Цель проекта
Закрепить знания двух спринтов и самостоятельно спроектировать
многоклассовое приложение с нуля — без подсказок и готового кода.

---

## ⚙️ Функциональность
- Ввод количества шагов за любой день любого месяца
- Изменение ежедневной цели по шагам (по умолчанию 10 000)
- Статистика за выбранный месяц:
    - шаги по дням
    - общее количество шагов
    - максимум за день
    - среднее за активные дни
    - пройденная дистанция в км
    - сожжённые килокалории
    - лучшая серия дней когда цель была достигнута

---

## 🏗 Архитектура — 4 класса

### `Main.java`
Точка входа. Консольное меню в цикле, обработка команд пользователя через `switch`.

### `StepTracker.java`
Основная логика приложения:
- хранит массив из 12 объектов `MonthData` (по одному на каждый месяц)
- принимает и валидирует ввод пользователя
- управляет целью шагов
- выводит статистику

### `MonthData.java`
Хранит и обрабатывает данные одного месяца:
- массив `int[] days` на 30 дней
- подсчет суммы, максимума, среднего
- алгоритм поиска лучшей серии

### `Converter.java`
Конвертация шагов:
- 1 шаг = 75 см → километры
- 1 шаг = 50 кал → килокалории

## 🧠 Что нового узнал

### Связь между классами
```java
// StepTracker хранит массив объектов MonthData
MonthData[] monthToData = new MonthData[12];

// В конструкторе инициализируем каждый объект
for (int i = 0; i < monthToData.length; i++) {
    monthToData[i] = new MonthData();
}

// Обращаемся к нужному месяцу по индексу
MonthData monthData = monthToData[month - 1];
monthData.days[day - 1] = steps;
```
Один объект может хранить массив других объектов.
Это основа ООП - объекты взаимодействуют друг с другом.

### Алгоритм лучшей серии
```java
public int bestSeries(int goalByStepsPerDay) {
    int currentSeries = 0;
    int bestSeries = 0;
    for (int i = 0; i < days.length; i++) {
        if (days[i] >= goalByStepsPerDay) {
            currentSeries++;
        } else {
            if (currentSeries > bestSeries) {
                bestSeries = currentSeries;
            }
            currentSeries = 0;
        }
    }
    return bestSeries;
}
```
Счётчик серии сбрасывается при пропуске дня.
Лучший результат сохраняется отдельно.

### Валидация ввода через while
```java
while (month < 1 || month > 12) {
    System.out.print("Введите номер месяца: ");
    month = scanner.nextInt();
    if (month < 1 || month > 12) {
        System.out.println("Такого месяца нет!");
    }
}
```
Цикл не пустит дальше пока пользователь не введёт корректное значение.

## 💥 С какими сложностями столкнулся

**Связь между классами** - самое сложное в этом проекте.
Когда смотришь уроки - всё понятно. Когда садишься делать сам -
непонятно как объекты должны передавать данные друг другу.
Решение пришло когда начал думать не о коде, а о логике:
StepTracker знает о месяцах, MonthData знает о днях - каждый
отвечает за своё.

**Scanner один на всё приложение** - сначала создавал новый
Scanner в каждом классе. Потом понял что правильнее создать
один в Main и передать через конструктор в StepTracker.

**Индексы массивов** - месяцы с 1 до 12, индексы с 0 до 11.
Дни с 1 до 30, индексы с 0 до 29. Везде `month - 1` и `day - 1`.
