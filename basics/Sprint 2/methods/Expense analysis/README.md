# 🐱🐹 Анализ расходов на корм с помощью методов

## 📋 Задача

Необходимо проанализировать расходы на корм для двух питомцев — кота **Пикселя** и хомяка **Байта**.

Для каждого питомца программа должна:

* 👋 Поздороваться;
* 💸 Найти стоимость самого дорогого корма;
* 📊 Посчитать общую сумму расходов;
* 🍽️ Пожелать приятного аппетита.

Главная цель задания — **декомпозировать код**, разделив его на небольшие методы, каждый из которых отвечает только за одну задачу.

---

## 🧠 Теория

### Декомпозиция

Вместо того чтобы писать всю логику внутри `main`, программа разбита на отдельные методы:

* `sayHello()` — приветствие питомца;
* `findMaxExpense()` — поиск самой дорогой покупки;
* `findExpensesSum()` — подсчёт общей суммы расходов;
* `sayEnjoyMeal()` — пожелание приятного аппетита.

Такой подход делает код более понятным, читаемым и удобным для повторного использования.

---

### Методы `void`

Если метод ничего не возвращает, используется `void`.

```java
public static void sayHello(String name)
```

Метод просто выполняет действие — печатает приветствие.

---

### Методы с возвращаемым значением

Если нужно получить результат вычислений, используется `return`.

```java
public static double findMaxExpense(double[] array)
```

Метод проходит по массиву расходов и возвращает максимальное значение.

Аналогично работает:

```java
public static double findExpensesSum(double[] array)
```

который суммирует все элементы массива и возвращает итоговую сумму.

---

### Передача параметров

Каждый метод принимает необходимые данные через параметры.

Например:

```java
findMaxExpense(feedExpensesCat);
```

Благодаря этому один и тот же код можно использовать как для кота, так и для хомяка, не дублируя логику.

---

## 🛠 Что использовал

* Методы (`void` и с возвращаемым значением)
* Параметры методов
* Оператор `return`
* Массивы `double[]`
* Цикл `for`
* Поиск максимального значения
* Подсчёт суммы элементов массива
* Декомпозицию программы

---

## 💡 Вывод

Это задание отлично показывает одну из главных идей программирования: **каждый метод должен выполнять только одну конкретную задачу**.

Когда логика разбита на небольшие независимые части, код становится проще читать, тестировать и дорабатывать. Именно такой подход используется в реальных проектах и позволяет создавать масштабируемые приложения без хаоса и дублирования.
