# 💰 Финансовый помощник

## 📋 Задача

Разработать программу, которая анализирует финансовый план пользователя и помогает понять, хватает ли зарплаты на все запланированные расходы.

Пользователь вводит:

* 💵 Размер зарплаты
* 🚌 Расходы на транспорт
* 🛒 Расходы на продукты
* 💰 Сумму, которую хочет отложить

Программа сравнивает доходы и расходы, а затем либо сообщает об успешном финансовом плане, либо предлагает скорректировать бюджет.

---

## 🧠 Теория

### Методы

Вся логика расчётов вынесена в отдельный метод:

```java
correctExpenses(salary, transportMoney, foodMoney, savings);
```

Такой подход делает код более структурированным и избавляет от дублирования.

---

### Передача параметров

Метод принимает четыре аргумента:

```java
public static void correctExpenses(
    int salary,
    int transportMoney,
    int foodMoney,
    int savings
)
```

Благодаря этому метод может работать с любыми входными данными, переданными из `main`.

---

### Локальные переменные

Внутри метода вычисляются:

```java
int expensesSum = transportMoney + foodMoney + savings;
int lackMoney = expensesSum - salary;
int leftMoney = salary - expensesSum;
```

* `expensesSum` — общая сумма расходов;
* `lackMoney` — сколько не хватает до выполнения плана;
* `leftMoney` — сколько останется свободных денег.

---

### Автоматическая корректировка бюджета

Если расходов оказалось больше зарплаты:

* программа постепенно уменьшает расходы на еду по 100 рублей;
* если этого недостаточно — полностью убирает запланированные накопления.

```java
while ((salary - foodMoney) < transportMoney) {
    foodMoney = foodMoney - 100;
}
```

После корректировки пользователь получает более реалистичный финансовый план.

---

## 🛠 Что использовал

* Методы (`public static void`)
* Передачу параметров
* Локальные переменные
* Условные конструкции `if/else`
* Цикл `while`
* Арифметические операции
* Ввод данных через `Scanner`

---

## 💡 Вывод

Это первое приложение, где основная логика вынесена в отдельный метод, а программа не просто выводит информацию, а самостоятельно анализирует ситуацию и предлагает решение.

По сути, это небольшой финансовый помощник, который показывает важную идею программирования: **сложную задачу проще решать, если разделить её на отдельные логические части и оформить их в виде методов.**
