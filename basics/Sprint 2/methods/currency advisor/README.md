# 💱 Советник по валюте

## 📋 Задача
Исправить ошибки в программе которая дает советы о покупке
долларов и евро. После совета программа снова запрашивает команду.

---

## 🧠 Теория

### Вынесение логики в отдельный метод
```java
public static void main(String[] args) {
    doCommand(); // вся логика в отдельном методе
}

public static void doCommand() {
    // логика здесь
}
```
main остается чистым - только вызов метода.
Вся логика инкапсулирована в doCommand().

### return vs break
```java
} else if (command == 0) {
    System.out.println("Выход");
    return; // выходим из метода целиком
}
```
`return` в void-методе - выходит из метода полностью.
`break` - только из цикла, метод продолжит работу дальше.

---

## 🛠 Что использовал
- Вынесение логики в отдельный метод `doCommand()`
- Бесконечный цикл `while (true)` с выходом через `return`
- `scanner.nextInt()` для считывания команды

---

## 💡 Вывод
Разбивай код на методы - main должен быть точкой входа,
а не местом где живет вся логика программы.
