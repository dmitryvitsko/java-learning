# 💰 Финансовое приложение v4.1.1

## 📋 Задача
Вынесение логики конвертации в отдельный класс Converter.
Создание объекта класса и вызов его метода из главного класса.

---

## 🧠 Теория

### Класс и объект
```java
// Класс - это шаблон
public class Converter {
    double rateUSD = 94.8;
    void convert(double rubles, int currency) { ... }
}

// Объект - это экземпляр класса
Converter converter = new Converter();
converter.convert(moneyBeforeSalary, currency);
```
Класс описывает что умеет делать объект.
Объект - это конкретный экземпляр с которым работаем.

### Вынесение логики в отдельный класс
Раньше весь код был в одном классе Practicum.
Теперь конвертация - в отдельном классе Converter.
Practicum просто вызывает `converter.convert()` не зная как это работает внутри.
Это называется инкапсуляция - один из принципов ООП.

### Методы в отдельных статических методах
```java
public static void printMenu() { ... }
public static void getAdvice(...) { ... }
public static double findMaxExpense(...) { ... }
```
Каждый метод делает одно дело. main() только управляет логикой.

## 🛠 Что использовал
- Создание объекта класса `new Converter()`
- Вызов метода объекта `converter.convert()`
- Разделение кода на статические методы
- Два отдельных класса в двух файлах
