# 🏗 Финансовый помощник — переход к объектно-ориентированному программированию v4.2.3

## 📋 Задача

Продолжить развитие финансового приложения и вынести работу с расходами в отдельный класс `ExpensesManager`.

Теперь программа состоит из нескольких самостоятельных классов:

* `Practicum` — взаимодействие с пользователем;
* `Converter` — конвертация валют;
* `DinnerAdvisor` — советы по расходам;
* `ExpensesManager` — управление расходами пользователя.

Главная цель задания — познакомиться с принципом разделения ответственности между классами.

---

## 🧠 Теория

### Почему понадобился новый класс?

До этого вся логика расходов находилась внутри основного класса программы.

По мере роста приложения код становился всё больше и сложнее для поддержки.

Поэтому работа с расходами была вынесена в отдельный класс:

```java
public class ExpensesManager
```

Теперь этот класс полностью отвечает за хранение и обработку расходов пользователя.

---

## 📦 Поля класса

Для хранения расходов используется поле:

```java
double[] expenses;
```

Массив содержит расходы за неделю.

Каждый индекс соответствует одному дню:

| Индекс | День        |
| ------ | ----------- |
| 0      | Понедельник |
| 1      | Вторник     |
| 2      | Среда       |
| 3      | Четверг     |
| 4      | Пятница     |
| 5      | Суббота     |
| 6      | Воскресенье |

---

## 🔨 Конструктор класса

При создании объекта автоматически создаётся массив на семь дней:

```java
public ExpensesManager() {
    expenses = new double[7];
}
```

Конструктор выполняется каждый раз при создании нового объекта:

```java
ExpensesManager expensesManager = new ExpensesManager();
```

---

## 💸 Метод `saveExpense()`

```java
double saveExpense(double moneyBeforeSalary,
                   int day,
                   double expense)
```

Метод выполняет несколько задач:

* уменьшает остаток денег;
* сохраняет трату за выбранный день;
* предупреждает пользователя при низком балансе;
* возвращает новый остаток средств.

Пример использования:

```java
moneyBeforeSalary =
        expensesManager.saveExpense(
                moneyBeforeSalary,
                day,
                expense
        );
```

---

## 📊 Метод `printAllExpenses()`

```java
void printAllExpenses()
```

Метод выводит все расходы за неделю.

Пример результата:

```text
День 1. Потрачено 500.0 рублей
День 2. Потрачено 750.0 рублей
День 3. Потрачено 300.0 рублей
...
```

Так как массив уже является полем класса, передавать его в параметры больше не требуется.

---

## 🔥 Метод `findMaxExpense()`

```java
double findMaxExpense()
```

Метод перебирает весь массив расходов и находит максимальную сумму.

Алгоритм работы:

1. Создать переменную `maxExpense`;
2. Пройти по массиву циклом `for`;
3. Сравнить каждое значение;
4. Вернуть максимальную найденную сумму.

Пример вызова:

```java
System.out.println(
    "Самая большая сумма расходов на этой неделе составила "
    + expensesManager.findMaxExpense()
    + " руб."
);
```

## 🏛 Разделение ответственности между классами

После рефакторинга каждый класс отвечает только за свою область:

### Practicum

* меню приложения;
* ввод данных пользователя;
* обработка команд.

### Converter

* конвертация валют по курсам.

### DinnerAdvisor

* рекомендации по расходам до зарплаты.

### ExpensesManager

* хранение расходов;
* добавление расходов;
* вывод расходов;
* поиск максимальной траты.

Такой подход называется **разделением ответственности (Single Responsibility Principle)**.

Теперь приложение состоит не из одного большого файла, а из нескольких специализированных классов, каждый из которых отвечает за свою задачу.
