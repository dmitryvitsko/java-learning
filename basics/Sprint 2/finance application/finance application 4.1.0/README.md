# 💰 Финансовый помощник с декомпозицией на методы

## 📋 Задача

Доработать финансовое приложение, разделив его на отдельные методы.

Программа позволяет:

* 💱 Конвертировать оставшиеся деньги в разные валюты;
* 🍕 Получать совет по расходам до зарплаты;
* 📝 Записывать ежедневные траты;
* 📊 Просматривать расходы за неделю;
* 🔥 Находить самый затратный день;
* 📋 Работать через удобное консольное меню.

Главная цель задания — научиться **разделять большую программу на независимые логические блоки**.

---

## 🧠 Теория

### Декомпозиция

Вместо того чтобы хранить всю логику внутри `main()`, программа разделена на отдельные методы.

Каждый метод отвечает только за одну задачу:

* `printMenu()` — выводит меню;
* `convert()` — конвертирует деньги в выбранную валюту;
* `getAdvice()` — даёт совет по расходам;
* `main()` — управляет работой приложения и связывает все части вместе.

Такой подход делает код значительно понятнее и удобнее для поддержки.

---

## 📋 Метод `printMenu()`

```java
public static void printMenu()
```

Метод не принимает параметры и ничего не возвращает (`void`).

Его единственная задача — вывести пользователю список доступных команд.

Это избавляет от повторения одинакового кода внутри цикла `while`.

---

## 💱 Метод `convert()`

```java
public static void convert(Scanner scanner, double moneyBeforeSalary)
```

Метод принимает:

* объект `Scanner` для ввода данных;
* текущий остаток денег.

Внутри метода происходит:

* выбор валюты;
* расчёт курса;
* вывод результата конвертации.

При этом все переменные, связанные с курсами валют, находятся внутри метода и не засоряют `main()`.

---

## 🍕 Метод `getAdvice()`

```java
public static void getAdvice(double moneyBeforeSalary, int daysBeforeSalary)
```

На основе остатка денег и количества дней до зарплаты программа рекомендует:

* экономить;
* заказать пиццу;
* сходить в кафе;
* позволить себе ресторан.

Логика полностью инкапсулирована внутри одного метода.

---

## 📊 Работа с массивом расходов

Для хранения трат используется массив:

```java
double[] expenses = new double[7];
```

Каждый элемент соответствует одному дню недели.

При добавлении новой траты:

* остаток денег уменьшается;
* расход сохраняется в соответствующей ячейке массива;
* при необходимости выводится предупреждение о низком балансе.

---

## 🔥 Поиск максимальной траты

Чтобы найти самую большую сумму расходов за неделю, программа проходит по всему массиву и сравнивает элементы между собой:

```java
if (expenses[i] > maxExpense) {
    maxExpense = expenses[i];
}
```

В результате пользователь сразу видит свой самый дорогой день.

## 💡 Вывод

Это задание показало, насколько сильно методы меняют архитектуру программы.

Функциональность практически не изменилась, но код стал значительно чище: каждая часть отвечает только за свою задачу, а `main()` превратился в понятный сценарий работы приложения.

Именно так строятся реальные проекты — не одной огромной функцией на сотни строк, а множеством небольших методов, каждый из которых решает одну конкретную проблему.
