# 💰 Финансовый помощник — полная декомпозиция приложения

## 📋 Задача

Продолжить рефакторинг финансового приложения и вынести обработку расходов в отдельные методы.

Теперь программа умеет:

* 💱 Конвертировать деньги в разные валюты;
* 🍕 Давать советы по расходам до зарплаты;
* 📝 Сохранять ежедневные траты;
* 📊 Показывать расходы за неделю;
* 🔥 Находить самую большую трату;
* 📋 Работать через удобное консольное меню.

Главная цель задания — сделать код максимально структурированным и избавиться от перегруженного метода `main()`.

---

## 🧠 Теория

### Что такое декомпозиция?

Декомпозиция — это разделение большой программы на небольшие независимые методы, каждый из которых отвечает только за одну конкретную задачу.

Вместо огромного `main()` логика распределяется между отдельными функциями:

* `printMenu()` — вывод меню;
* `convert()` — конвертация валют;
* `getAdvice()` — рекомендации по расходам;
* `saveExpense()` — сохранение новой траты;
* `printAllExpenses()` — вывод всех расходов;
* `findMaxExpense()` — поиск максимальной траты.

Благодаря этому код становится проще читать, тестировать и дорабатывать.

---

## 💾 Метод `saveExpense()`

```java
public static double saveExpense(Scanner scanner,
                                 double moneyBeforeSalary,
                                 double[] expenses)
```

Метод отвечает за сохранение новой траты.

Он:

* спрашивает день недели;
* считывает сумму расхода;
* уменьшает текущий баланс;
* записывает расход в массив;
* предупреждает пользователя при низком остатке средств;
* возвращает обновлённый баланс.

Использование `return` позволяет сохранить новое значение переменной `moneyBeforeSalary`.

---

## 📊 Метод `printAllExpenses()`

```java
public static void printAllExpenses(double[] expenses)
```

Метод проходит по массиву расходов и выводит все траты по дням недели.

Для перебора используется цикл `for`.

Пример вывода:

```
День 1. Потрачено 500.0 рублей
День 2. Потрачено 1200.0 рублей
...
```

---

## 🔥 Метод `findMaxExpense()`

```java
public static void findMaxExpense(double[] expenses)
```

Метод перебирает весь массив расходов и ищет максимальное значение.

Алгоритм простой:

1. Создаётся переменная `maxExpense`;
2. Последовательно сравнивается каждый элемент массива;
3. Если найдено большее значение — оно сохраняется;
4. После завершения цикла выводится самая большая трата за неделю.

Такой подход позволяет быстро анализировать расходы пользователя.

---

## 💱 Конвертация валют

Метод `convert()` принимает текущий баланс и объект `Scanner`, после чего позволяет пересчитать сумму в:

* 🇺🇸 USD
* 🇪🇺 EUR
* 🇨🇳 CNY

Все курсы валют инкапсулированы внутри метода и не загромождают основной код программы.

---

## 🍕 Получение финансового совета

Метод `getAdvice()` анализирует:

* количество оставшихся денег;
* число дней до зарплаты.

На основании этих данных приложение рекомендует:

* экономить;
* заказать пиццу;
* сходить в кафе;
* позволить себе ресторан.

## 💡 Вывод

После очередного этапа рефакторинга программа стала значительно чище.

Теперь `main()` практически не содержит бизнес-логики — он только координирует работу приложения, а вся функциональность распределена по отдельным методам.

Именно такой подход используется в профессиональной разработке: каждая функция решает одну задачу, а вместе они образуют понятную, масштабируемую и легко поддерживаемую программу.
