# 💰 Финансовое приложение v4.2.2

## 📋 Задача
Добавлен конструктор с параметрами в класс Converter.
Курсы валют теперь передаются при создании объекта, а не захардкожены в классе.

---

## 🧠 Теория

### Конструктор класса
```java
public class Converter {
    double rateUSD;

    Converter(double usd, double eur, double cny) {
        rateUSD = usd; // инициализация полей при создании объекта
    }
}
```
Конструктор - специальный метод который вызывается при создании объекта.
Имя совпадает с именем класса. Нет возвращаемого типа.

### Конструктор по умолчанию
```java
// Если не написать конструктор - Java создает его сама:
Converter() {} // конструктор по умолчанию

// Но если написать свой конструктор с параметрами -
// конструктор по умолчанию перестает генерироваться автоматически!
// Поэтому Converter() без параметров больше не работает.
```

### Передача параметров при создании объекта
```java
// Было (поля захардкожены внутри класса):
Converter converter = new Converter();

// Стало (значения передаются снаружи):
Converter converter = new Converter(94.8, 103.8, 13.1);
```
Теперь можно создать конвертер с любыми курсами - класс стал гибким.

---

## 🛠 Что использовал
- Конструктор с параметрами `Converter(double usd, double eur, double cny)`
- Инициализация полей класса через конструктор
- Передача аргументов при создании объекта `new Converter(94.8, 103.8, 13.1)`

---

## 💡 Вывод
Конструктор с параметрами делает класс гибким - можно создавать
объекты с разными начальными значениями.
Это основа ООП: данные и логика работы с ними хранятся вместе в классе.
